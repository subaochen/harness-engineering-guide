# 文章插图生成规范

> **版本**: 1.0  
> **生效时间**: 2026-04-21  
> **适用范围**: 所有对外发布的文章

---

## 🎯 核心规则

**所有对外发布的文章，插图必须使用专业工具生成，确保专业性和一致性。**

---

## 🛠️ 推荐工具

### 专业插图工具（优先级排序）

| 工具 | 用途 | 推荐指数 | 说明 |
|------|------|----------|------|
| **baoyu-image-gen** | AI 图像生成 | ⭐⭐⭐⭐⭐ | 支持多模型，专业提示词 |
| **baoyu-cover-image** | 封面图生成 | ⭐⭐⭐⭐⭐ | 5 维度定制，公众号友好 |
| **baoyu-infographic** | 信息图生成 | ⭐⭐⭐⭐⭐ | 数据可视化，专业图表 |
| **baoyu-comic** | 知识漫画 | ⭐⭐⭐⭐ | 教程类文章适用 |
| **baoyu-article-illustrator** | 文章配图 | ⭐⭐⭐⭐ | 自动匹配文章内容 |

### 不推荐工具

| 工具 | 原因 |
|------|------|
| 简单截图 | 不够专业，缺乏设计感 |
| 随手拍照 | 质量不稳定，不专业 |
| 网络图片 | 版权问题，风格不统一 |
| 低质量 AI 生成 | 细节粗糙，影响专业度 |

## 🎨 插图标准

### 技术指标

| 指标 | 要求 | 说明 |
|------|------|------|
| **分辨率** | ≥1920x1080 | 高清显示 |
| **格式** | JPG/PNG | 通用格式 |
| **文件大小** | <5MB | 便于加载 |
| **宽高比** | 16:9 / 4:3 | 标准比例 |
| **色彩模式** | RGB | 屏幕显示 |

### 设计规范

| 规范 | 要求 | 说明 |
|------|------|------|
| **风格统一** | 同一篇文章使用相同风格 | 保持视觉一致性 |
| **配色协调** | 使用统一配色方案 | 推荐科技蓝/紫色系 |
| **文字清晰** | 如有文字，确保可读性 | 避免模糊文字 |
| **主题相关** | 插图与内容强相关 | 避免无关装饰 |
| **版权清晰** | 使用原创或授权图片 | 避免版权纠纷 |

## 📋 插图类型

### 1. 封面图

**用途**：文章首页/列表页展示

**要求**：
- ✅ 分辨率：1920x1080（16:9）
- ✅ 包含文章主题视觉元素
- ✅ 留白区域用于标题文字
- ✅ 风格专业，吸引点击

**生成命令**：
```bash
bun ~/.openclaw/skills/baoyu-cover-image/scripts/main.ts \
  --promptfile article-cover-prompt.md \
  --image cover.jpg \
  --ar 16:9 \
  --quality 2k
```

---

### 2. 架构图

**用途**：系统架构、流程说明

**要求**：
- ✅ 清晰的层次结构
- ✅ 统一的图标风格
- ✅ 文字标注清晰
- ✅ 配色专业

**生成工具**：
- baoyu-infographic（信息图）
- 专业绘图工具（Draw.io/Excalidraw）

---

### 3. 流程图

**用途**：工作流程、操作步骤

**要求**：
- ✅ 流程清晰，箭头明确
- ✅ 步骤编号清晰
- ✅ 统一图标风格
- ✅ 配色协调

**生成工具**：
- baoyu-infographic
- Draw.io / Mermaid

---

### 4. 数据图表

**用途**：数据展示、对比分析

**要求**：
- ✅ 数据准确
- ✅ 图表类型合适
- ✅ 坐标轴标注清晰
- ✅ 配色专业

**生成工具**：
- baoyu-infographic
- Chart.js / D3.js

---

### 5. 代码截图

**用途**：代码示例

**要求**：
- ✅ 使用代码高亮工具
- ✅ 字体清晰可读
- ✅ 背景色协调
- ✅ 避免直接截图 IDE

**推荐工具**：
- Carbon (https://carbon.now.sh/)
- CodeSnap (VS Code 插件)
- Markdown 代码块（GitHub 自动高亮）

## 🚀 标准流程

### Step 1：确定插图需求

**在写作时标记**：
```markdown
<!-- 插图需求：封面图 - AI Agent 协作系统架构图 -->
<!-- 插图需求：流程图 - 事务处理流程 -->
<!-- 插图需求：数据图表 - 性能对比 -->
```

### Step 2：生成提示词

**创建提示词文件**：
```bash
cat > article-cover-prompt.md << 'EOF'
# 文章封面图提示词

## 文章信息
- 标题：XXX
- 主题：XXX
- 关键词：XXX

## 设计要求
- 类型：conceptual
- 配色：warm
- 渲染：flat-vector
- 文字：title-only
- 情绪：balanced
- 宽高比：16:9

## 视觉元素
- 元素 1：XXX
- 元素 2：XXX
- 元素 3：XXX
EOF
```

### Step 3：生成插图

**使用专业工具**：
```bash
# 封面图
bun ~/.openclaw/skills/baoyu-cover-image/scripts/main.ts \
  --promptfile article-cover-prompt.md \
  --image cover.jpg \
  --ar 16:9 \
  --quality 2k

# 信息图
bun ~/.openclaw/skills/baoyu-infographic/scripts/main.ts \
  --prompt "系统架构图，3 层结构..." \
  --image architecture.png \
  --ar 4:3
```

### Step 4：质量检查

**检查清单**：
- [ ] 分辨率≥1920x1080
- [ ] 文件大小<5MB
- [ ] 风格与文章一致
- [ ] 配色协调
- [ ] 文字清晰可读
- [ ] 主题相关性强
- [ ] 无版权问题

### Step 5：保存到指定目录

**目录结构**：
```
articles/
└── article-name/
    ├── cover.jpg              # 封面图
    ├── images/
    │   ├── architecture.png   # 架构图
    │   ├── workflow.png       # 流程图
    │   └── chart.png          # 数据图表
    └── article.md             # 文章
```

### Step 6：文章中引用

**Markdown 引用**：
```markdown
![封面图：AI Agent 协作系统](./cover.jpg)

![架构图](./images/architecture.png)

![流程图](./images/workflow.png)
```
